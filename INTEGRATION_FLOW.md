# Процесс интеграции классификатора консонансности

## Архитектура интеграции

```
┌─────────────────────────────────────────────────────────────────┐
│                        Web Interface                             │
│  (HTML/CSS/JavaScript - отображение консонансной оценки)        │
└─────────────────┬───────────────────────────────────────────────┘
                  │
                  │ POST /api/generate
                  ▼
┌─────────────────────────────────────────────────────────────────┐
│                    Flask Backend (app.py)                        │
│  1. Получает FASTA последовательность                            │
│  2. Генерирует MIDI через BioMusicGenerator                     │
│  3. Вызывает ConsonanceClassifier.classify()                    │
│  4. Возвращает результат с консонансной оценкой                 │
└─────────────────┬───────────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────────┐
│         ConsonanceClassifier (consonance_classifier.py)          │
│                                                                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 1. Lazy load PyTorch & CRNN model                       │   │
│  │    - Загрузка происходит только при первом использовании│   │
│  │    - Кэширование модели в памяти                       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                          │                                       │
│                          ▼                                       │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 2. Extract MIDI features                                │   │
│  │    - midi_path_to_melspec() из crnn_music_classifier   │   │
│  │    - Меллограммы для каждого трека + микс              │   │
│  └─────────────────────────────────────────────────────────┘   │
│                          │                                       │
│                          ▼                                       │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 3. CRNN Model Inference                                 │   │
│  │    - SharedCNN: обработка каждого трека                │   │
│  │    - TrackAttention: агрегация информации о треках     │   │
│  │    - GRU: временная суммаризация                        │   │
│  │    - FC Classifier: consonant/dissonant                │   │
│  └─────────────────────────────────────────────────────────┘   │
│                          │                                       │
│                          ▼                                       │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 4. Return Predictions                                   │   │
│  │    - prediction: "consonant" | "dissonant"             │   │
│  │    - confidence: 0.0-1.0                                │   │
│  │    - scores: {consonant: float, dissonant: float}      │   │
│  └─────────────────────────────────────────────────────────┘   │
└─────────────────┬───────────────────────────────────────────────┘
                  │
                  ▼ (consonance object in response)
┌─────────────────────────────────────────────────────────────────┐
│                  Frontend Display (app.js)                       │
│                                                                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ Consonance Rating Card:                                 │   │
│  │                                                          │   │
│  │  Предсказание:  [CONSONANT / DISSONANT]               │   │
│  │  Уверенность:   87%                                     │   │
│  │                                                          │   │
│  │  ▓▓▓▓▓▓▓▓▓▓░░░ Консонансность  87%                     │   │
│  │  ░░░░░░░░░░░░░ Диссонансность  13%                     │   │
│  └─────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

## Пример API запроса и ответа

### Запрос:
```bash
curl -X POST http://localhost:5001/api/generate \
  -H "Content-Type: application/json" \
  -d '{
    "fasta": ">Example\nATCGATCGATCG..."
  }'
```

### Ответ:
```json
{
  "success": true,
  "session_id": "abc12345",
  "header": "Example",
  "sequence_length": 100,
  "consonance": {
    "success": true,
    "prediction": "consonant",
    "confidence": 0.8723,
    "scores": {
      "consonant": 0.8723,
      "dissonant": 0.1277
    }
  },
  "musical_params": {
    "tempo": 120.5,
    "key": "C major",
    "sequence_type": "dna",
    "harmony_bars": 25,
    "melody_notes": 156,
    "num_fragments": 4
  },
  "audio_available": true,
  "audio_filename": "abc12345.wav"
}
```

## Обработка ошибок

### Классификатор не готов:
```json
{
  "consonance": {
    "success": false,
    "error": "Model not found at /path/to/model.pth"
  }
}
```

### Генерация MIDI успешна, но классификация не удалась:
```json
{
  "consonance": {
    "success": false,
    "error": "Classification failed: ..."
  }
}
```
В этом случае генерация композиции всё равно успешна, классификация просто пропускается.

## Тестирование интеграции

### 1. Проверка готовности классификатора:
```bash
curl http://localhost:5001/api/status
```

Ответ должен содержать:
```json
{
  "consonance_classifier": {
    "ready": true,
    "error": null
  }
}
```

### 2. Генерация с консонансной оценкой:
- Используйте веб-интерфейс для генерации композиции
- Проверьте, что отобразилась секция "Анализ консонансности"
- Убедитесь, что показывается предсказание и уверенность

### 3. Мониторинг производительности:
- Первая генерация (с загрузкой модели): ~5-10 секунд
- Последующие генерации: ~2-3 секунды (зависит от длины MIDI)
