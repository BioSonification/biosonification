## Конфигурация данных

### FASTA

Основной FASTA-файл задаётся полем `fasta_path` в конфиге:

```json
{
  "fasta_path": "data/fasta/refseq_genomes/GCF_000005845.2_genomic.fna"
}
```

Поддерживаются расширения: `.fasta`, `.fa`, `.fna`, `.ffn`, `.faa`, `.frn`

Энкодер принимает DNA, RNA и protein-like последовательности, очищает записи и режет длинные последовательности на фрагменты по параметрам из `bio`.

### MIDI

Полифонический MIDI-корпус задаётся списком `music.midi_dirs`:

```json
{
  "music": {
    "midi_dirs": [
      "data/midi/POP909",
      "data/midi/maestro-v3.0.0"
    ]
  }
}
```

Корпус должен содержать файлы: `.mid`, `.midi`, `.xml`, `.mxl` или `.musicxml`

Из них извлекаются структурированные сегменты:
- Гармония по тактам
- Монофоническая мелодическая линия
- Музыкальные дескрипторы для pairing

**Примечание:** Встроенный `music21` fallback подходит для smoke-тестов. Для серьезных экспериментов используйте внешний лицензированный полифонический MIDI-корпус.

## Генерация отчета по данным

Перед обучением рекомендуется зафиксировать, какие данные реально попали в запуск:

```powershell
.\.venv\Scripts\python.exe tools\report_structured_dataset.py `
  --config configs\pipeline_v2_medium_rtx2060_fast.json `
  --output-dir results\v2_dataset_report
```

Команда создаёт:
- `results/v2_dataset_report/dataset_report.json`
- `results/v2_dataset_report/dataset_report.md`

Отчёт показывает:
- Количество FASTA records/fragments
- Количество MIDI-файлов
- Тип музыкального источника
- Число извлечённых structured-сегментов
- Распределения tempo/key/density
- Фактические параметры сегментации

## Типичный рабочий процесс

1. **Подготовка FASTA:**
   - Положите FASTA в `data/fasta/` или укажите абсолютный путь в конфиге

2. **Подготовка MIDI:**
   - Положите полифонические MIDI/XML-файлы в отдельный каталог
   - Обновите `music.midi_dirs` в конфиге

3. **Создание конфигурации:**
   - Создайте копию `configs/pipeline_v2_medium_rtx2060_fast.json`
   - Обновите `fasta_path` и `music.midi_dirs`

4. **Генерация отчета:**
   ```powershell
   .\.venv\Scripts\python.exe tools\report_structured_dataset.py --config your_config.json --output-dir results\dataset_report
   ```

5. **Обучение:**
   ```powershell
   .\.venv\Scripts\python.exe train_bio_music_v2.py --config your_config.json
   ```

6. **Генерация музыки:**
   ```powershell
   .\.venv\Scripts\python.exe generate_from_fasta_v2_fragmented.py `
     --checkpoint results\v2_medium_rtx2060_fast\checkpoints\structured_pipeline.pt `
     --fasta your_sequence.fna `
     --output output.mid
   ```

## Частые проблемы

**MIDI-файлы не находятся:**
- Проверьте `music.midi_dirs` и расширения файлов
- Убедитесь, что пути корректны

**Сегментов слишком мало:**
- Увеличьте корпус
- Ослабьте ограничения: `music.min_notes_per_segment`, `music.bars_per_segment`, `music.max_segments`

**FASTA не читается:**
- Проверьте, что заголовки начинаются с `>`
- Убедитесь, что последовательность содержит допустимые символы

**Результаты трудно воспроизвести:**
- Сохраните конфиг, `dataset_report.*`, `metrics.json` и checkpoint
- Runtime-каталоги (`results/`, `outputs/`, `tmp/`, `web/output/`) не хранятся в git
